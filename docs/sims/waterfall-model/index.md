## About This MicroSim

The classic five-phase waterfall (Requirements → Design → Implementation → Verification → Maintenance) drawn as a stair-step cascade. Hover any phase for activities, artifacts, and the most common stall. Toggle **Show Feedback Arrows** to switch from "pure waterfall" to the "modified waterfall" most teams actually use.

## Lesson Plan

### Learning Objectives

By the end of this activity, students will be able to:

1. Describe each waterfall phase
2. Identify the primary risk in each phase
3. Articulate when waterfall is an appropriate method
