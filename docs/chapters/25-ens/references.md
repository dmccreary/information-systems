# References: The Enterprise Nervous System

1. [Event-driven architecture](https://en.wikipedia.org/wiki/Event-driven_architecture) - Wikipedia - Comprehensive overview of EDA patterns, message brokers, and event sourcing. Foundation for the chapter's ENS architectural framing.

2. [Complex event processing](https://en.wikipedia.org/wiki/Complex_event_processing) - Wikipedia - Detailed coverage of CEP, pattern detection over event streams, and major engines. Anchors the chapter's real-time-rules content.

3. [Apache Kafka](https://en.wikipedia.org/wiki/Apache_Kafka) - Wikipedia - Coverage of the dominant streaming platform's architecture, partitioning, and consumer groups. Directly supports the chapter's streaming-platform content.

4. Designing Event-Driven Systems - Ben Stopford - O'Reilly - Concise reference on event-driven architectures with Kafka; directly aligned with this chapter's depth and apply-level outcomes.

5. Streaming Systems - Tyler Akidau, Slava Chernyak, and Reuven Lax - O'Reilly - The definitive reference on stream processing semantics including watermarks, windowing, and exactly-once delivery; a load-bearing source for the chapter's streaming content.

6. [Apache Kafka Documentation](https://kafka.apache.org/documentation/) - Apache Software Foundation - Official documentation for the dominant streaming platform that backs most ENS implementations.

7. [Apache Flink Documentation](https://nightlies.apache.org/flink/flink-docs-stable/) - Apache Software Foundation - Reference for the leading open-source stream-processing engine for stateful event processing and CEP.

8. [Confluent Streaming Tutorials](https://developer.confluent.io/learn/) - Confluent - Free educational content on streaming platforms, schema management, and stream processing patterns. Excellent practitioner supplement.

9. [AWS EventBridge Documentation](https://docs.aws.amazon.com/eventbridge/) - Amazon Web Services - Reference for cloud-native event-bus services, useful for grounding the abstract ENS architecture in concrete implementations.

10. [Reactive Manifesto](https://www.reactivemanifesto.org/) - Reactive Foundation - Foundational principles of reactive systems (responsive, resilient, elastic, message-driven) that directly inform the chapter's ENS design philosophy.
