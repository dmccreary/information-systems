# References: Enterprise Systems

1. [Enterprise resource planning](https://en.wikipedia.org/wiki/Enterprise_resource_planning) - Wikipedia - Comprehensive overview of ERP history, modules, vendors (SAP, Oracle, Workday), and implementation challenges. Anchors the chapter's ERP content.

2. [Customer relationship management](https://en.wikipedia.org/wiki/Customer_relationship_management) - Wikipedia - Detailed treatment of CRM systems including Salesforce, Microsoft Dynamics, and HubSpot. Foundation for the chapter's CRM section.

3. [Supply chain management](https://en.wikipedia.org/wiki/Supply_chain_management) - Wikipedia - Coverage of SCM concepts, software categories, and integration with ERP. Supports the chapter's enterprise-systems landscape MicroSim.

4. Concepts in Enterprise Resource Planning (4th Edition) - Ellen Monk and Bret Wagner - Cengage - The dominant academic ERP textbook; provides the SAP-grounded examples and process-integration content this chapter draws from.

5. SAP S/4HANA: An Introduction (5th Edition) - Devraj Bardhan, Axel Baumgartl, et al. - SAP Press - Practitioner reference on the modern ERP stack from the dominant vendor; useful for grounding abstract ERP discussions in concrete implementations.

6. [SAP Help Portal](https://help.sap.com/) - SAP - Authoritative documentation for the largest ERP vendor's products. Useful when students want concrete examples of ERP module structure.

7. [Salesforce Trailhead](https://trailhead.salesforce.com/) - Salesforce - Free interactive learning platform for the dominant CRM. Excellent hands-on supplement for the chapter's CRM content.

8. [Workday Customer Center](https://www.workday.com/en-us/products/human-capital-management/overview.html) - Workday - Vendor information on the leading cloud HRIS. Concrete reference for the chapter's HRIS and human capital management content.

9. [ERP Implementation Lifecycle](https://www.panorama-consulting.com/erp-implementation-process/) - Panorama Consulting - Practitioner guide to ERP implementation phases, common failure modes, and cutover strategies. Reinforces the chapter's implementation timeline MicroSim.

10. [Configuration vs Customization](https://www.gartner.com/en/information-technology/glossary/configuration) - Gartner - Industry-analyst framing of the configuration-vs-customization tradeoff that defines so much of enterprise-systems work. Directly supports the chapter's customization decision content.
